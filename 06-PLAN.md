### Key APIs & code patterns

**Current `getBundleExamStats` signature** (`src/lib/services/bundle.ts:106`):
```ts
export async function getBundleExamStats(db: Db, bundleId: number)
```
Returns: `{ exams, attempts, totalAttempts, completedAttempts, avgScore, bestScore, worstScore, totalTimeSeconds }`. Currently includes ALL attempts; unfinished ones get `score: 0` in averages.

**Current `getBundleCardWeakness` signature** (`src/lib/services/bundle.ts:183`):
```ts
export async function getBundleCardWeakness(db: Db, bundleId: number)
```
Returns: `{ card, total, incorrect, correct, incorrectRate }[]`. Joins answers through `examAttempts` but does **not** filter on `completedAt` — includes answers from unfinished attempts.

**Unfinished attempt**: `attempt.completedAt === null` (set in `examAttempts` table). No score, no duration.

**shadcn Switch**: Installed via `pnpm dlx shadcn@latest add switch`. Renders as `<Switch />` from `@/components/ui/switch`. Uses `radix-ui` `SwitchPrimitive.Root`. Usage: `<Switch checked={value} onCheckedChange={setValue} />`.

**Checkbox pattern** (existing, `src/components/ui/checkbox.tsx`): imports `Checkbox as CheckboxPrimitive` from `"radix-ui"`, uses `data-slot` convention and `cn()` utility. The Switch component will follow the same conventions when generated by shadcn CLI.

**Stats page data flow** (`src/app/(main)/study-dome/bundles/[id]/stats/page.tsx`):
- Fetches `getBundleExamStats(db, bundleId)` and `getBundleCardWeakness(db, bundleId)` in `useEffect`
- Derives `attemptPoints`, `donutData`, `weaknessData`, `totalCorrect`, `totalIncorrect` via `useMemo`
- Currently assigns `score: 0` for unfinished attempts in `attemptPoints`
- Summary cards show: Total Attempts, Average Score, Best Score, Total Time

**Past-exams filter pattern** (`src/app/(main)/study-dome/bundles/[id]/past-exams/page.tsx`):
- Uses `<Select>` dropdown with `statusFilter` state (`"all"` | `"completed"` | `"unfinished"`)
- `filteredAttempts` derived via `useMemo` — same pattern we'll use for the Switch toggle

### Gotchas
- The project uses `radix-ui` (unified package), NOT individual `@radix-ui/*` packages. The shadcn CLI should generate Switch using the unified import.
- React Compiler is enabled (`reactCompiler: true` in `next.config.ts`). Do NOT add `useMemo`/`useCallback` where the compiler handles it — but the existing code already uses them, so maintain consistency with the surrounding patterns.
- `examAttempts.completedAt` is `null` for unfinished, `Date.now()` integer for completed. Score is `null` for unfinished, `0–1` float for completed.
- `getBundleCardWeakness` uses `INNER JOIN` on `examAttempts` — adding a `completedAt IS NOT NULL` filter requires adding it to the `WHERE` clause with `and()`.
- The `pnpm dlx shadcn@latest add switch` command must be used (project uses pnpm, not npm/npx).
- Stats page already has `completedAttempts` in the returned stats object (line 138 in `bundle.ts`). We can leverage this.

## Phase 1 — Service Layer: Add `skipUnfinished` Option

### Task 1.1: Add `skipUnfinished` option to `getBundleExamStats`

**What**: Add an optional `options` parameter to `getBundleExamStats` that, when `{ skipUnfinished: true }`, filters out unfinished attempts from all calculations and the returned `attempts` array.

**Files**: `src/lib/services/bundle.ts`

**Implementation notes**:
1. Change the function signature from:
   ```ts
   export async function getBundleExamStats(db: Db, bundleId: number)
   ```
   to:
   ```ts
   export async function getBundleExamStats(db: Db, bundleId: number, options?: { skipUnfinished?: boolean })
   ```
2. After fetching `attempts`, apply filtering BEFORE computing stats:
   ```ts
   const skipUnfinished = options?.skipUnfinished ?? false;
   const effectiveAttempts = skipUnfinished
     ? attempts.filter((a) => a.attempt.completedAt != null)
     : attempts;
   ```
3. Use `effectiveAttempts` instead of `attempts` for:
   - `allScores` calculation
   - `totalAttempts` count
   - `completedAttempts` count
   - `totalTimeSeconds` calculation
4. Return `effectiveAttempts` as the `attempts` field (so the UI can also use the filtered list for charts).
5. Keep `exams` unchanged (exam definitions are not affected by the filter).
6. When `skipUnfinished` is `false` or omitted, behavior is identical to current (backward compatible).

**Tests**: Add to `src/lib/services/__tests__/bundle.test.ts`

- **Test: `skipUnfinished` excludes unfinished attempts from stats**:

  4. Call `getBundleExamStats(db, bundleId, { skipUnfinished: true })`.
  5. Assert `totalAttempts === 1`, `completedAttempts === 1`, `avgScore === 1.0`, `bestScore === 1.0`, `worstScore === 1.0`.
  6. Assert `attempts.length === 1` (only the completed attempt).

  4. Assert `attempts.length === 2`.

### Task 1.2: Add `skipUnfinished` option to `getBundleCardWeakness`

**What**: Add an optional `options` parameter to `getBundleCardWeakness` that, when `{ skipUnfinished: true }`, excludes answers from unfinished exam attempts.

**Files**: `src/lib/services/bundle.ts`

**Implementation notes**:
1. Change the function signature from:
   ```ts
   export async function getBundleCardWeakness(db: Db, bundleId: number)
   ```
   to:
   ```ts
   export async function getBundleCardWeakness(db: Db, bundleId: number, options?: { skipUnfinished?: boolean })
   ```
2. When `skipUnfinished` is `true`, add `sql\`${schema.examAttempts.completedAt} IS NOT NULL\`` to the `WHERE` clause of both the `totalAnswers` query (line 195–210) and `incorrectAnswers` query (line 213–228):
   ```ts
   const skipUnfinished = options?.skipUnfinished ?? false;
   const completedFilter = skipUnfinished
     ? sql\`${schema.examAttempts.completedAt} IS NOT NULL\`
     : undefined;

   // In totalAnswers query:
   .where(
     and(
       eq(schema.exams.bundleId, bundleId),
       inArray(schema.examAnswers.cardId, cardIds),
       sql\`${schema.examAnswers.isCorrect} IS NOT NULL`,
       completedFilter,
     )
   )

   // In incorrectAnswers query:
   .where(
     and(
       eq(schema.exams.bundleId, bundleId),
       inArray(schema.examAnswers.cardId, cardIds),
       eq(schema.examAnswers.isCorrect, false),
       completedFilter,
     )
   )
   ```
3. When `skipUnfinished` is `false` or omitted, behavior is identical to current (backward compatible).

**Tests**: Add to `src/lib/services/__tests__/bundle.test.ts`

- **Test: default includes answers from unfinished attempts**:

   ```ts
         const { db } = await getDb();
   ```

   ```tsx

     stats && (stats.totalAttempts > 0 || donutData.length > 0 || weaknessData.length > 0);
   ```

   This already works because `totalAttempts` will be 0 if all attempts were filtered out. But add a helpful message when there are only unfinished attempts:

7. The `attemptPoints` useMemo currently maps unfinished attempts to `score: 0`. After the service-layer filter, when `skipUnfinished` is true, unfinished attempts won't appear in `stats.attempts` at all. Update the `attemptPoints` memo to handle both cases:
   ```ts
   const attemptPoints = useMemo<AttemptPoint[]>(() => {
     if (!stats) return [];
     return stats.attempts.map((a, i) => ({
       index: i + 1,
       date: new Date(a.attempt.startedAt).toLocaleDateString(),
       score: Math.round((a.attempt.score ?? 0) * 100),
       status: a.attempt.completedAt != null ? "completed" : "unfinished",
     }));
   }, [stats]);
   ```
   Since `a.attempt.score` is `null` for unfinished but the `?? 0` fallback handles it, this code is already correct. When `skipUnfinished` is true, no unfinished attempts will be in the array, so the `status` field will always be `"completed"`.

8. In the summary cards, update the "Total Attempts" card subtitle to reflect filtering:
